# Project Plan - Sprint 2

**Project:** Factory Inventory & Production Management System (MVP)  
**Role:** Project Manager  
**Sprint:** 2  
**Version:** 1.0

## 1. Sprint Objective
Deliver the full core MVP baseline with operational workflows usable for factory-level demonstration.

## 2. Planned Deliverables
- Role-based authentication integration (Firebase + backend role mapping).
- CRUD modules for products, suppliers, wholesalers, and users.
- GRN draft/confirm flow with stock-in updates.
- BOM and batch lifecycle (draft/start/complete).
- Order management, FEFO dispatch, delivery note PDF.
- Reports and audit query surface.

## 3. Scope Governance
- MVP-first: avoid nonessential analytics and custom billing features.
- Keep endpoint contracts stable to reduce UI rework.
- Prioritize production/stock accuracy over visual polish.

## 4. Acceptance Conditions
- End-to-end operational flow executable with seeded data.
- Core modules reachable via role-appropriate UI menus.
- No unresolved high-severity blocker in stock mutation flows.

## 5. Sprint 2 Outcome
Planned scope largely delivered, with deployment and production runtime concerns transferred to Sprint 3.
